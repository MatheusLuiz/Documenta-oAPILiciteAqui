Listar Logins de Usuários

Para listar todos os logins de usuários disponíveis, utilize o seguinte endpoint:

  • GET /listar: Lista todos os logins de usuários.

Cadastrar Login de Usuário

Para cadastrar um novo login de usuário, envie uma solicitação POST contendo os seguintes parâmetros no corpo da requisição:

  • usuario: Inteiro - ID único do usuário (campo obrigatório).
  • username: String - Nome de usuário para login (campo obrigatório).
  • senha: String - Senha do usuário para acesso ao sistema (campo obrigatório).
  • usuario_log: Inteiro - ID do usuário responsável pela criação do cadastro (campo obrigatório).

Atualizar Login de Usuário

Para atualizar um registro existente na tabela Login Usuários, envie uma solicitação PUT contendo os seguintes parâmetros no corpo da requisição:

  • id_login: Inteiro - ID único do login que será atualizado (campo obrigatório).
  • username: String - Nome de usuário para login (campo obrigatório).
  • senha: String - Nova senha do usuário para acesso ao sistema.
  • usuario_log: Inteiro - ID do usuário responsável pela atualização do cadastro (campo obrigatório).

Deletar Login de Usuário

Para deletar um registro existente na tabela Login Usuários, envie uma solicitação DELETE contendo os seguintes parâmetros no corpo da requisição:

  • id_login: Inteiro - ID único do login que será deletado (campo obrigatório).
  • usuario_log: Inteiro - ID do usuário responsável pela exclusão do registro (campo obrigatório).

Autenticar Usuário

Para autenticar um usuário existente, envie uma solicitação POST contendo os seguintes parâmetros no corpo da requisição:

  • username: String - Nome de usuário (campo obrigatório).
  • senha: String - Senha do usuário (campo obrigatório).

Este endpoint retornará os dados do usuário autenticado (exceto a senha), caso o login seja bem-sucedido.
