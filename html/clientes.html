Listar Clientes

Para listar os clientes, utilize os seguintes endpoints sem a necessidade de parâmetros adicionais:

  • GET /listar/ativos: Lista todos os clientes ativos.
  • GET /listar/inativos: Lista todos os clientes inativos.
  • GET /listar/suspensos: Lista todos os clientes suspensos.
  • GET /listar/todos: Lista todos os clientes.

Cadastrar Cliente

Para cadastrar um Cliente, envie uma solicitação POST contendo os seguintes parâmetros no corpo da requisição:

  • cnpj: String - CNPJ do cliente (14 dígitos).
  • razao_social: String - Razão social do cliente.
  • nome_fantasia: String - Nome fantasia do cliente.
  • status: String - Status do cliente (1 para ativo, 0 para inativo).
  • data_cadastro: String - Data de cadastro no formato YYYY-MM-DD.
  • usuario: String - ID do usuário responsável pelo cadastro.

Atualizar Cliente

Para atualizar um registro existente na tabela Clientes, envie uma solicitação PUT contendo os seguintes parâmetros no corpo da requisição:

  • id_cliente: String - ID único do cliente que será atualizado (campo obrigatório).
  • cnpj: String - CNPJ do cliente (14 dígitos).
  • razao_social: String - Razão social do cliente.
  • nome_fantasia: String - Nome fantasia do cliente.
  • status: Inteiro - Status do cliente (1 para ativo, 0 para inativo).
  • usuario: Inteiro - ID do usuário que está realizando a atualização.

Deletar Cliente

Para deletar um registro da tabela Clientes, envie uma solicitação DELETE contendo os seguintes parâmetros no corpo da requisição:

  • id_cliente: Inteiro - ID único do cliente que será deletado (campo obrigatório).
  • usuario: Inteiro - ID do usuário que está realizando a exclusão.
